## Remaining findings — all informational (ℹ️)

Nothing rises to Critical/Warning anymore. These are accept-or-optional:

1. **Single content page (topical depth).** Only `/` + `/disclaimer.html`. This is the one real
   *traffic* lever left — one evergreen supporting page (e.g. "Net vs total carbs") would widen
   keyword surface and add internal links. **Optional**: it bends the no-touch constraint, so it's a
   deliberate choice, not a defect. Everything else on-page is done.

2. **Lab LCP 6.9s** — a late-painting AdSense unit under throttling. Field LCP is 2.1s (green).
   "Fixing" it means shrinking an ad. **Don't** — it trades revenue for an already-green field metric.

3. **`twitter:site` / `twitter:creator` missing** — optional, and only addable if there's an X/Twitter
   handle to point at. Cards already render fine via `summary_large_image`.

4. **Other AI crawlers not explicitly listed** (Applebot-Extended, CCBot, Bytespider, anthropic-ai,
   Amazonbot). They inherit the permissive `*` so they're already allowed; list them only if you want
   to explicitly allow (visibility) or block (e.g. Bytespider/CCBot). Low priority.

5. **FAQPage JSON-LD won't earn rich results** (restricted to gov/health-authority sites since Aug
   2023). Keep it — still parsed for AI answers. No action.

6. **HSTS `max-age=0` + missing security headers** (CSP, X-Frame-Options, …). Not a ranking factor,
   and GitHub Pages can't set custom response headers. Note only.

7. **Images are JPG/PNG, not WebP/AVIF.** Field CWV is green and lab TBT is 0, so no LCP problem —
   skip unless a future audit shows an image-driven regression.
