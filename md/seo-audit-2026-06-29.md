## Findings (by impact)

### 🔴 / ⚠️ High impact

**1. Render-blocking Google Fonts request (~780ms).** *(Confirmed via PageSpeed)*
- *Evidence:* Field CWV all pass (LCP 2.1s / INP 143ms / CLS 0.09), so ranking-wise CWV is fine.
  The one flagged render-blocking resource is the Google Fonts stylesheet (`fonts.googleapis.com/css2…`,
  1.4 KiB, 780ms) — it blocks first paint. Lab FCP 3.3s / LCP 8.4s are throttled-simulation only.
- *Impact:* Removing the render block improves FCP/LCP for the slowest users (and the lab score)
  without changing appearance — `&display=swap` already shows fallback text immediately.
- *Fix (one-time):* Load the fonts stylesheet non-blocking:
  ```html
  <link rel="preload" as="style" href="…css2…">
  <link rel="stylesheet" href="…css2…" media="print" onload="this.media='all'">
  <noscript><link rel="stylesheet" href="…css2…"></noscript>
  ```
  Heavy third-party JS (Charts/AdSense/jQuery) shows as lab TBT 490ms but field INP is 143ms — real
  interactivity is fine, so no JS surgery needed.

**2. No author `Person` schema on a YMYL (health) page.** ⚠️
- *Evidence:* JSON-LD has `WebApplication` + publisher `Organization`, but no `author`/`Person`
  with `sameAs`. Author is named in body text only.
- *Impact:* This is a YMYL topic; E-E-A-T applies to all competitive queries (Dec 2025). A machine-
  readable author with external profiles strengthens trust signals for Google + AI engines.
- *Fix (one-time):* Add a `Person` (name "Martin Leitner-Ankerl", `sameAs` → blog, reddit, GitHub)
  and reference it as `author` of the page. Keep the honest "not a medical professional" framing.

**3. Single content page — thin topical surface / internal linking.** ⚠️
- *Evidence:* Only real pages are `/` and `/disclaimer.html`. Internal links point outward (reddit,
  wikipedia, ncbi, amazon), not to supporting on-site content.
- *Impact:* Limits keyword coverage and topical authority — the main *traffic* ceiling. **But** it
  directly tensions the no-touch constraint.
- *Fix (optional, one-time):* If you ever want the traffic lever, add **one** evergreen supporting
  page (e.g. "Keto macros explained" or "Net vs total carbs") and link it from the FAQ. One page,
  written once. Otherwise accept the single-page scope as a deliberate trade-off.

### ⚠️ Medium impact

**4. No `llms.txt` — missed AI-search/GEO surface.** *(Confirmed: HTTP 404)*
- *Impact:* AI engines (ChatGPT, Perplexity, Claude) increasingly use `llms.txt` to summarize a
  site. Evergreen, zero-maintenance — ideal for a static site. The visible FAQ already feeds AEO well.
- *Fix:* Add `/llms.txt`: site name, one-line description, links to `/` and `/disclaimer.html`.

**5. `og:description` too long (230 chars).** *(Confirmed)*
- *Impact:* Truncated in social previews. *Fix:* trim to ≤200 chars (reuse the meta description).

**6. AI crawlers not explicitly managed in robots.txt.** *(Confirmed)*
- *Evidence:* GPTBot, ClaudeBot, PerplexityBot, Google-Extended etc. inherit the permissive `*` —
  i.e. they **are** currently allowed (good for visibility), just not explicit.
- *Fix:* For AI-search visibility, add explicit `User-agent: GPTBot` / `Allow: /` blocks (signals
  intent; pairs with llms.txt). Only block them if you'd rather *not* be in AI answers.

### ℹ️ Low impact / informational

**7. `FAQPage` JSON-LD won't earn rich results.** Since Aug 2023 FAQ rich results are restricted to
gov/health-authority sites; a commercial calculator won't get the accordion. **Keep it anyway** —
it's still parsed for AI Overviews/AEO and the visible FAQ is genuinely useful. No action needed.

**8. OG polish:** missing `og:site_name`, `og:locale`, and `og:image:width`/`height` (image is
1600px — declare dimensions for crisp rendering). Trivial, one-time.

**9. Insecure `http://` outbound links (~25):** wikipedia, ncbi, amazon, reddit links use `http://`.
They redirect to HTTPS so it's cosmetic, but updating to `https://` once removes the redirect hop and
looks cleaner. Low priority.

**10. HSTS `max-age=0` + missing security headers (CSP, X-Frame-Options, etc.).** Security posture,
not a ranking factor — and GitHub Pages can't set custom response headers anyway. Note only; not an
SEO action.

**11. Images not next-gen format.** JP(E)G/PNG, no WebP/AVIF. Minor LCP opportunity if #1 shows the
share/warning images matter; otherwise skip (they're lazy-loaded or below the fold).
